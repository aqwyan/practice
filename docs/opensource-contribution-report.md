## Процесс внесения вклада

### 1. Выбор проекта
Я выбрала проект «Дока», потому что это активное русскоязычное сообщество, которое создаёт понятную документацию для веб-разработчиков. Проект имеет более 1 400 звёзд на GitHub, что соответствует требованию задания (50+ звёзд).

### 2. Поиск задачи
Я изучила структуру репозитория и решила добавить интерактивную демку для статьи о CSS свойстве `display`. В статье не хватало наглядного примера, который помог бы новичкам понять разницу между значениями `block`, `inline`, `inline-block`, `flex` и `grid`.

### 3. Создание контента
Я написала интерактивную демку на HTML/CSS/JS, которая позволяет:
- Выбирать значение `display` из выпадающего списка
- Мгновенно видеть, как меняется расположение блоков
- Читать пояснения к каждому значению

### 4. Внесение изменений через GitHub
- Сделала Fork репозитория `doka-guide/content`
- Через веб-интерфейс GitHub отредактировала файл `css/display/index.md`
- Добавила код демки в тело статьи
- Создала коммит с описанием изменений

### 5. Создание Pull Request
- Оформила Pull Request с подробным описанием
- Указала, какие изменения были сделаны
- Отправила PR на рассмотрение редакторам проекта
